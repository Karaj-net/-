# Лекция 5
## Задачи линейного программирования (ЛП) и методы их решения. Двойственная задача ЛП, анализ решения на чувствительность и устойчивость.

Задачи линейного программирования представляют собой важный класс оптимизационных задач, в которых как целевая функция, так и ограничения являются линейными.

### Графический метод решения задачи ЛП

Графический метод основан на геометрической интерпретации задачи ЛП. Из-за визуальных ограничений он в основном применяется для решения двухмерных задач (где количество переменных $n = 2$). 

Графический метод состоит из 2 основных этапов:
1. **Построение области допустимых решений.** По линейным ограничениям задачи ЛП строится многогранник (или многоугольник) решений. 
2. **Нахождение оптимальной точки.** Строится целевая функция и градиент к ней. Линия уровня целевой функции смещается по направлению градиента к крайней точке многоугольника решений. 

> *Примечание:* Возможны случаи, когда область допустимых решений (многоугольник) получается неограниченной.

### Симплекс-метод решения задач ЛП

Универсальным и самым распространенным методом решения задачи ЛП является **симплекс-метод**. Он непосредственно применяется к задаче ЛП, записанной в *канонической форме*. 

Каждое базисное решение (опорный план) определяет координаты вершины многогранника решений. Из этих вершин необходимо выбрать ту, которая доставляет минимум или максимум целевой функции $f(x)$. 

При больших числах переменных $n$ и ограничений $m$ найти оптимальное решение простым перебором всех базисных решений очень трудно, так как число переборов будет равняться числу сочетаний из $n$ по $m$ ($C_n^m$). Поэтому необходим метод упорядоченного перехода от одного базисного решения к другому. Таким алгоритмом и является симплекс-метод.

> **Теорема:** Если существует оптимальное решение задачи ЛП, оно может быть получено с помощью симплекс-метода, причем алгоритм можно начинать с любого исходного базиса.

Решение задачи ЛП симплекс-методом состоит из 2 этапов:
1. Определение первоначального (исходного) базисного решения.
2. Последовательное улучшение исходного базисного решения до оптимального за счет перемены местами одной базисной переменной с другой небазисной (один шаг приближает систему к оптимуму).

Симплекс-метод позволяет последовательно перебирать базисные решения за конечное число шагов. На каждом шаге проверяется, является ли текущее базисное решение оптимальным для минимизации/максимизации функции:
$$\min f(x) = \sum C_i X_i$$

Обычно при ручном счете вычисления по симплекс-методу проводятся с использованием специальных **симплекс-таблиц**.

### Двойственная задача ЛП

Иногда вместо исходной задачи ЛП выгоднее решать задачу, двойственную к ней. Это особенно актуально, когда число ограничений исходной задачи гораздо больше числа её переменных.

Пусть исходная (прямая) задача минимизации записана в виде:
$$\min f(x) = \sum C_i X_i$$

Тогда двойственная к ней задача будет формулироваться на максимум функции $f_i(y)$ со следующей структурой ограничений:
$$\max f_i(y) = \sum B_j X_j$$
При условиях-ограничениях:
$$\sum A_{ij} Y_j \le C_i, \quad i = 1, \dots, n$$

Таким образом, всегда имеется возможность выбора: решать двойственную или основную задачу. Выбирается тот вариант, который проще реализуется вычислительно.

> **Теорема:** Если исходная задача имеет решение, то двойственная задача также имеет решение, причем экстремумы их целевых функций совпадают:
$$\min f(x) = \max f_i(y)$$

### Анализ на чувствительность и устойчивость полученного решения

После нахождения оптимального решения задачи ЛП в обязательном порядке необходимо провести математический анализ на чувствительность и устойчивость полученного плана к изменениям исходных параметров (коэффициентов целевой функции, ограничений по ресурсам и технологических коэффициентов матрицы).
