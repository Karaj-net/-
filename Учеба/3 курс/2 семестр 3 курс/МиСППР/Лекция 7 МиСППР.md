# Лекция 7
## Градиентные методы решения НЛП (продолжение). Безградиентные методы. Методы сканирования. Метод случайного поиска. Выпуклое программирование.

Существует целая группа методов решения задач нелинейного программирования (НЛП) под названием **безградиентные методы**. В процессе поиска оптимального решения в них используется только информация, полученная в результате сравнительной оценки критериев оптимальности при выполнении каждого шага (вычисление производных не требуется).

### Метод сканирования

Метод заключается в последовательном просмотре значений целевой функции в ряде точек из допустимой области исходных значений. Среди них находится точка, в которой достигается экстремум. Точность определения зависит от густоты расположения выбранных точек допустимой области. В результате сравнения всех значений определяется точка экстремума.

* **Достоинства:** предельная простота алгоритма; метод гарантирует нахождение *глобального экстремума*.
* **Недостатки:** необходимость вычисления целевой функции в очень большом количестве точек.

Для устранения недостатка существуют различные модификации метода сканирования. Например, сначала функция вычисляется с большим (грубым) шагом, а затем в окрестностях наилучшей точки шаг уменьшается для локального уточнения.

### Симплекс-метод Нелдера — Мида (кильдра нида)

Основная идея данного безградиентного метода заключается в том, что по известным значениям целевой функции в вершинах выпуклого многогранника (который называется *симплексом*) определяется направление, в котором требуется выполнить следующий шаг, чтобы получить уменьшение целевой функции. 

Главным процессом здесь является **отражение** — нахождение вершины нового симплекса, расположенного симметрично относительно плоскости, проходящей через одну из сторон исходного симплекса. Выбор направления поиска новой вершины определяется положением той вершины исходного симплекса, значение целевой функции в которой оказалось наихудшим. Известен также более сложный модифицированный симплекс-метод, в котором при отражении происходит масштабирование (сжатие или растяжение) симплекса.

### Специальные классы задач НЛП

1. **Сепарабельное программирование** — группа задач, в которых целевая функция может быть представлена в виде суммы функций, каждая из которых зависит только от одной переменной:
   $$f(x) = \sum_{i=1}^{n} f_i(x_i)$$
2. **Метод геометрического программирования** — применяется для решения задач инженерного характера, когда целевая функция и ограничения заданы в виде положительных полиномов (почтичленов):
   $$f(x) = \sum_{j=1}^{m} C_j \prod_{i=1}^{n} x_i^{a_{ji}} \cdot c_j$$
   В основе методов решения задач геометрического программирования лежат свойства геометрических неравенств (например, связь среднего арифметического и среднего геометрического: $\frac{a+b}{2} \ge \sqrt{ab}$).
3. **Метод неопределенных множителей Лагранжа** — классический метод, который применяется для решения задач, когда все ограничения заданы строгими равенствами:
   $$\max f(x) \quad \text{при условии } \phi_j(x) = 0, \ j = 1, \dots, m$$

### Методы случайного поиска

Данные методы аналогичны методу сканирования (копирования), но точки выбираются не по регулярной сетке, а случайным образом.

* **Метод слепого поиска:** из допустимой области случайным образом выбирается точка, в которой вычисляется значение целевой функции. Это значение сравнивается с предыдущим наилучшим результатом. Система запоминает точку, которая ближе всего подобралась к оптимальной.
* **Метод случайных направлений:** движение к экстремуму осуществляется по формуле шага, где направление задается случайным вектором $a_k$:
   $$x_{k+1} = x_k + h_k \cdot a_k$$
* **Метод $\psi$-преобразования:** один из эффективных методов, позволяющих решать многоэкстремальные (глобальные) задачи НЛП. Объектом исследования является не сама исходная целевая функция $f(x)$, а некоторая преобразованная функция $\phi$. Если оригинал $f(x)$ является измеримой функцией, определенной на некотором множестве или терпящей разрывы первого рода, то преобразованная функция $f(L)$ является монотонно убывающей и строго соответствует значению глобального экстремума исходной целевой функции $f(x)$.

### Выпуклое и квадратичное программирование

Множество точек образует *выпуклое множество* в пространстве, если для любых двух точек этого множества отрезок прямой линии, соединяющий их, полностью принадлежит данному множеству.

Задача выпуклого программирования записывается в виде:
$$\max(\min) f(x), \quad x(x_1, x_2, x_3) \quad \text{при } \phi_j(x) \le \beta_i, \ j = 1, \dots, m$$
где целевая функция $f(x)$ является вогнутой (при максимизации) или выпуклой (при минимизации), а функции ограничений $\phi_j(x)$ — выпуклыми. 

> **Главное свойство выпуклого программирования:** для таких задач любой *локальный экстремум* автоматически является *глобальным*.

#### Теорема Куна — Таккера
Для решения задач выпуклого программирования вводится **функция Лагранжа** (где $Y_n$ — множество множителей Лагранжа). 

Согласно теории Куна — Таккера, если множество допустимых решений обладает свойством регулярности, то точка является оптимальной тогда и только тогда, когда существует такой вектор множителей Лагранжа, что выполняются условия, при которых данная точка является **седловой точкой** функции Лагранжа.

Частным случаем задач выпуклого программирования является **задача квадратичного программирования**, в которой ограничения линейны, а целевая функция содержит квадратичную форму (сумму парных произведений переменных):
$$f(x) = \sum_{j=1}^{n} a_j x_j + \sum_{i=1}^{n} \sum_{j=1}^{n} C_{ij} x_i x_j$$
где двойная сумма представляет собой отрицательно или положительно полуопределенную квадратичную форму.
