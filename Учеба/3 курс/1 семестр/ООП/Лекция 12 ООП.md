# Анонимные методы и лямбда-выражения 
Иногда методы используются только один раз, например, при инициализации делегатов, в таких случаях нет необходимости делать или описывать отдельный метод можно соответствующие тело прописать непосредственно в коде, использующий этот метод. Описанные таким образом методы называются - анонимные
```C#
delegate(параметры)
{
	body
}

Action del5 = delegate 
				{
					Console.Writeline("Анонимный метод");
				};
```
Как мы видим, если параметр отсутствует, то его можно опустить
```C#
Func <int, int, int> dues = delegate(int x, int y) {return x - y;};
```
Рассмотрим лямбда выражения. Если тело в одну строку, то не пишут фиг скобки, если только return, то не пишем return 
```C#
параметры => тело

//пример
Action del7 = () => 
{
	Console.Writeline("Лямбда");
	Console.Writeline("выражение");
}

Func <int, int, int>  del8 = (x, y) => x-y;
```
Анонимные методы и лямбда выражения могут использоваться в делегатов на ровне со всеми. Один и тот же делегат может одновременно указывать на несколько методов. В начале делегат инициализируется обычным образом, потом другим методы добавляются при помощи += в результате другие методы прописываются в списке вызова данного делегата и при выполнении делегата выполняются все методы из этого списка
```C#
Action del9 = new Test.F1;
	del9 += Test.F2;
	del9+= F3;
	del9 += new Test().F1;
	del9 += delegate{Console.Writeline("Анонимный");};
	del9 += () => Console.Writeline("Лямбда");
	del9();
	
Action del10 = new Test().F1;
Action del11 = Test.F2;
Action del12 = F3;
Action del13 = del10 + del11 + del12;
del13();
```
Если указанный метод есть в списке, то он удаляется, если нету, то операция игнорируется, если два, то удаляется тот, что ближе к концу
Иногда при удалении методов возникает опасность, что мы удалим все методы и получим значение Null - пустой, при попытке выполнить этот делегат получим ошибку, поэтому после выполнения желательно проверить, что мы не имеем значение null

Рассмотрим добавление функций, когда соответствующие методы что-то возвращают, в этом случае тоже все функции все включаются из списка, только отличие заключается в том, что выполнzются тела этих функций за исключением оператора return, этот оператор выполняется только для последней функции 
Рассмотрим пример:
```C#
Func <int, int, int> del14 = G1;
	del14 += G2;
	int a = del14(3, 8);
Console.Writeline(a.ToString());

int G1(int x, int y)
{
	Console.Writeline("Сумма");
	return x + y;
}

static int G2(int x, int y)
{
	Console.Writeline("Произведение");
	return x * y;
}
```
Мы получаем только return

# Использование делегатов при описании метода
Как мы знаем при описании методов определяются типы параметров и тип возвращаемого значения. Делегаты могут использоваться и в том и в другом случае, использование делегата для определения типа параметра некоторые параметры являются параметром, следовательно значение этого параметра будут какие-то методы 
```C#
void H(int x, int y, Func<int, int, int> d) => 
Console.Writeline("First {0}, second {1}, result {2}", x, y, d(x, y));
H(5, 10, G1);
H(5, 10, G2);
```
Когда делегаты используются для возращения типа. Здесь мы получаем в зависимости от значения параметров, результатом является тот или иной метод определенного вида
Рассмотрим примеры:
```C#
Action P(int x)
{
	switch(x)
	{
		case 1: return F3;
		case 2: return F2;
		default: return new Test().F1
	}
}

Action del15 = P(3)


Func<int, int, int> S(int k)
{
	if (k > 10)
		return G1;
	else
		return G2;
}
Console.Writeline(S(5)(7, 8));
```
