# Коллекции
## Коллекция Dictionary<K, L>
Это коллекция соответствует словарям. В данном случае наш объект, входящий в коллекцию состоит из ключа и объекта, в данном случае эта коллекция аналогична списку. В списке мы могли получить доступ по номеру, здесь по ключу. Номер является целым пол числом, он может изменяться при удалении, вставке и т.д. Ключ же может принимать любое значение и не меняется ни при каких действиях. Таким образом, во время создания экземпляра нужно сделать по значению и по ключу, можно создать экземпляр словаря по аналогии создания двумерного массива в качестве значений мы будем использовать экземпляры класса студент, а в качестве ключа название вуза
```C#
Student st1 = new Student("Иввнов", 5, 4, 5);

Student[] stnd = {st1, ..., st4};
Dictionary<string, Student> d1 = new Dictionary<string, Student>
{
	["КГАСУ"] = st1,
	["КФУ"] = st2,
	["КЮИ"] = st3 
}

d1.Add("МГПУ", st4); //Добавление нового элемента
d1.Remove("КЮИ"); //Удаление по ключу

foreach(var x in d1)
{
	Console.Writeline("Student {0}, for {1}", x.Value.Name, x.Key)
}

Console.Writeline(d1.ContainsValue(st5)));

Console.Writeline(d1.ContainsKey("КГАСУ"));
```
При помощи метода Add() можно добавить объект с двумя значениями. 
Можно проверить наличие конкретного объекта или конкретного ключа, для этого используется соответствует методы ContainsValue или ContainsKey()

## Понятие делегатов
Мы описывали классы разных типов и каждый класс определяет, делегат это тоже некоторый аналог класса он определяет функциональный тип значения его экземпляров являются разные методы, делегат именно определяет какой сигнатурой могут является его значения, заметим, что слово делегат здесь применяется в двух смыслах, как сам класс и как его экземпляр 
delegate тип имя (перемены);
```C#
delegate void Del();
```
Теперь нам надо описать методы соответствующих типов
Распишем 3 метода 
1. Статический - запишем в классе Program
2. Статист

```C#

{
	delegate void Del();
	Class Test
	{
		public void F1() => Console.Writeline("Динамический");
		public static void F2() => Console.Writeline("Статический");

```

## Встроенные делегаты 
Имеются делегаты, которые встроенные в фреймворк, рассмотрим некоторые из них
1. Делегат Action<T1, T2, ...> - это обобщенный делегат, у него до 16 параметров они определяют типы входных параметров. Этот делегат ничего не возвращает. Таким образом, описание данного делегата - выглядит так: delegate void Action<T1, t2>(T1 x1, T2 x2, ...)
2. Delegate Func<T1, T2, ...> - здесь также 16 параметров, последний определяет, выходящее значение, остальные - параметры
3. Делегат Predictect<T> - имеет один входной параметр и возвращает bool
